# GitHub Repository Setup

Use this checklist after pushing the portfolio release commit to GitHub. It makes the repository sidebar, topics, and release section look complete for recruiters and reviewers.

## Repository

- Owner/repo: `JasonTM17/Laptopshop_SpringBoot_MVC`
- Visibility: Public
- Default branch: `master`
- Website: leave blank until a real live demo URL exists

## About Sidebar

Set the GitHub repository description to:

```text
Spring Boot MVC laptop e-commerce portfolio app with polished storefront, customer checkout, real admin dashboard metrics, demo data, tests, Docker, CI, and deployment-ready configuration.
```

Set topics to:

```text
spring-boot spring-mvc spring-security jsp jstl mysql h2-database ecommerce docker github-actions portfolio java
```

Do not set a fake homepage URL. A blank Website field is better than a broken or circular link.

## Release

Create the first public release from the pushed portfolio commit.

| Field | Value |
| --- | --- |
| Tag | `v1.0.0` |
| Target | The commit containing the portfolio UI/docs/test updates |
| Title | `Laptopshop v1.0.0 Portfolio Showcase` |
| Assets | Source code zip/tar generated by GitHub only |

Release body:

```markdown
## Laptopshop v1.0.0 Portfolio Showcase

This release packages Laptopshop as a complete Spring Boot MVC portfolio project: polished storefront, real customer checkout flow, admin operations, demo data, automated tests, Docker/Render deployment assets, and reviewer-friendly documentation.

### Storefront

- Retail red Laptopshop UI with category navigation, promo sections, dense product cards, and responsive layout.
- Catalog search, brand/need/price filters, active chips, sorting, and pagination.
- Product detail with promotion, warranty/support, specs, related products, add-to-cart, and buy-now shortcut.
- Browser-local wishlist and search autocomplete.
- Professional About page with service story, release status, reviewer checklist, terms, and privacy.

### Customer Flow

- Demo customer account seeded in the `local` profile.
- Cart quantity and stock validation.
- Checkout validation and transactional order creation.
- Stock decrement, sold-count update, cart clearing, and order history.

### Admin Flow

- Real dashboard metrics through backend services.
- Revenue and factory charts, low-stock list, and recent activity.
- Backend search/filter for products, orders, and users.
- Validated order status transitions.
- CSV export at `/admin/report/orders.csv`.

### Backend, Security, And Deployment

- API `401/403` responses return JSON for unauthenticated/forbidden API calls.
- Clear not-found behavior for missing resources.
- Profile-based local/prod configuration with environment variables.
- Docker, Docker Compose, Render blueprint, health endpoints, sitemap, robots, and manifest.

### Documentation

- README with screenshots, routes, demo accounts, setup, tests, and portfolio notes.
- Installation, architecture, testing, deployment, reviewer guide, feature matrix, showcase script, and release notes.

### Verification

Validated locally:

```powershell
.\mvnw.cmd test
.\mvnw.cmd package
```

Browser smoke covered:

```text
/ -> /products -> /products?factory=APPLE -> /product/1
/login -> /cart -> /checkout -> /order-history
/about -> /admin
```

### Known Limits

- No live payment gateway in this release.
- Wishlist is browser-local, not account-synced.
- Product specs and promotion labels are derived from existing data/view rules.
- Public demo seed is intended for portfolio review, not production data.
```
